category: Added
- **VCS provider abstraction layer** - Complete platform abstraction for all PR and CI operations. The `internal/vcs.Provider` interface now covers PR lifecycle (create, merge, status, list), review comments (fetch, resolve threads), CI checks (fetch checks, failure logs), bot review comment fetching, and merged PR listing. GitHub implementation delegates to `ghpr` and `gh` CLI. All consumers (daemon, bellows, crucible, cifix, reviewfix) route operations through the VCS provider. `ParseRepoURL` now supports non-GitHub hosts (GitLab, Gitea, Bitbucket, Azure DevOps). Per-anvil `platform` config field selects the provider. (Forge-m89a)
