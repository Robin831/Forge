category: Added
- **Multiple generic webhook targets in notifications** - Added `notifications.webhooks[]` config to send a uniform JSON payload (`event_type`, `bead_id`, `anvil`, `message`, `timestamp`) to any HTTP endpoint. Each target can filter events independently. The Teams webhook keeps its Adaptive Card format; generic targets receive the simpler payload. Added `release` event type for generic webhooks alongside the existing `release_published` Teams event. Generic targets now correctly honour the global `notifications.enabled` flag, and receive `bead_decomposed` and `daily_cost` events (dispatched when a bead is decomposed by Schematic or when the daily cost limit is reached). (Forge-9fpc)
